# Hướng dẫn đóng góp cho Action Bird

Chào mừng bạn đến với dự án Action Bird! Chúng tôi đánh giá cao sự đóng góp của bạn để trò chơi ngày càng hoàn thiện hơn.

## Làm thế nào để đóng góp?

### 1. Báo lỗi (Reporting Bugs)
Nếu bạn phát hiện lỗi, vui lòng kiểm tra xem lỗi đó đã được báo cáo chưa tại mục [Issues]. Nếu chưa, hãy tạo một Issue mới và cung cấp các thông tin sau:
- Mô tả chi tiết lỗi.
- Các bước để tái hiện lỗi.
- Ảnh chụp màn hình hoặc log lỗi (nếu có).

### 2. Đề xuất tính năng mới
Chúng tôi luôn sẵn sàng lắng nghe các ý tưởng mới! Hãy tạo một Issue mới và gắn nhãn `feature request`.

### 3. Gửi Pull Request (PR)
Để gửi mã nguồn mới:
1. **Fork** dự án về tài khoản GitHub của bạn.
2. Tạo một **Branch** mới cho tính năng hoặc bản sửa lỗi của bạn (`git checkout -b feature/AmazingFeature`).
3. **Commit** các thay đổi (`git commit -m 'Add some AmazingFeature'`).
4. **Push** lên Branch (`git push origin feature/AmazingFeature`).
5. Mở một **Pull Request** tới nhánh `main` của dự án gốc.

## Quy tắc mã nguồn (Code Style)
- Sử dụng thụt lề 4 khoảng trắng (Standard Python).
- Viết chú thích (Comments) rõ ràng cho các hàm và lớp mới.
- Standard hóa các file tài nguyên theo cấu trúc thư mục `assets/`.

## Liên hệ
Nếu có bất kỳ thắc mắc nào, bạn có thể tạo Issue hoặc liên hệ qua trang quản lý của [TwinMindWeekly](https://github.com/TwinMindWeekly).

Cảm ơn bạn đã đồng hành cùng chúng tôi!
